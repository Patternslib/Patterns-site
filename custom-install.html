Getting Started With a Patterns Project

This is the easiest way to set up a maintainable project with patterns.

Using the make commands described here will make updating a patterns project really easy and will tightly integrate into your workflow. You can update patternslib in your project without affecting your custom styles.

Prerequisites

You need to have the following dependencies on your system before you proceed:

GIT: Download Git

NodeJS: Download NodeJS

Ruby 1.9+: Download Ruby

Create a new project

Compass project

  • Install compass: gem install compass
  • Clone compass project template
  • run npm install && bower install
  • grunt

LibSass project

  • Install grund: sudo npm install -g grunt-cli
  • Clone libsass project template
  • run bower install
  • bundle exec jekyll serve --watch

Jekyll project

  • bundle install to install jekyll
  • Clone jekyll project template
  • run bower install
  • bundle exec jekyll serve --watch

Update your project

There comes the point in time where you want to update the patterns version in your project. This is done in two simple steps:

  1. Specify the version you want to update to in your bower.json file.
  2. In your project folder, run 'bower update'

This will fetch the specified version of patterns from bower and install it into your project. The position will be the same as the earlier version you had so that all your includes should continue to work.

Now check if you need to update your custom styling and get going with the new patterns!

Use a Development Version of Patterns

Say you want to work on your project and on patterns at the same time and always have the latest checkout of patterns available. Then you don't want to wait for new releases but actually use the checkout of patterns directly in your project.

You can do that with a simple trick. By linking a checkout of patterns into your project. We use a bower mechanism for that.

In your project, do 'make link'. It will checkout a Patterns repository into your home directory parallel to your project directory and link that into your project where the patterns library would be fetched to. If there already is a Patterns repository, it will reuse that.

Now you can work within the patterns repository and all changes will be immediately available to your project site. Actually, the patterns website is built with this.
